# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Screens in scope: coach dashboard (lead list, lead profile, draft queue scaffold), integration health card, admin dashboard at /admin, auth screens (invite-only).

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui |
| Preset | default — customized per this spec (warm tones, glass surfaces) |
| Component library | Radix UI (via shadcn) |
| Icon library | @phosphor-icons/react — stroke weight 1.5, consistent across all surfaces |
| Font | Geist Sans (headings + body) + Geist Mono (sequence positions, timestamps, status codes) |
| Animation | Framer Motion — spring physics, layout transitions, staggered list reveals |
| Motion standard | `type: "spring", stiffness: 120, damping: 18` for interactive elements |

**Source:** CLAUDE.md (Next.js 15, Tailwind v4, shadcn/ui, Framer Motion); design-taste-frontend skill (Geist, Phosphor, spring physics).

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gap, badge inner padding |
| sm | 8px | Compact row spacing, form field gap |
| md | 16px | Default element spacing, card inner padding (horizontal) |
| lg | 24px | Card inner padding (vertical), section spacing within a panel |
| xl | 32px | Between major dashboard sections |
| 2xl | 48px | Page-level top/bottom padding |
| 3xl | 64px | Sidebar width unit, empty-state vertical breathing room |

Exceptions:
- Touch targets: minimum 44px tall for all interactive rows and buttons (accessibility minimum)
- Keyboard shortcut badge: 28px height (non-standard, single-use component)
- Timeline connector line: 2px width (visual only, not spacing)

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 600 (semibold) | 1.2 | Page titles (Dashboard, Admin, Lead name in profile header) |
| Heading | 20px | 600 (semibold) | 1.25 | Section headers, card titles, modal headings |
| Body | 16px | 400 (regular) | 1.5 | Lead notes, draft full text, description copy, table cells |
| Label | 14px | 400 (regular) | 1.4 | Form labels, metadata (state badge, source tag, timestamp), nav items |

Cap body line length at 65ch maximum (enforced on draft text and coach notes fields).

Monospace (Geist Mono) used for: touchpoint position indicator ("Message 3 of 6"), scheduled send timestamps in draft card, sequence status debug in admin view. Size: 13px, weight 400.

**Source:** design-taste-frontend skill (Geist, hierarchy rules); impeccable skill (typography laws).

## Color

### Philosophy

Strategy: **Committed** — one warm dominant tone carries the surface. Not restrained neutrals. Not tech-bro dark. Think: a warm, unhurried morning — the color of a coach's home office, not a trading terminal.

Both light and dark modes are fully supported. The glass/frosted card treatment bridges the two.

### Light Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(97% 0.008 60)` — warm off-white (warm cream-white) | Page background, app shell |
| Secondary (30%) | `oklch(94% 0.010 55)` — warm light sand | Cards, sidebar surface, nav background |
| Accent (10%) | `oklch(62% 0.14 50)` — warm amber-terracotta | Reserved for: primary CTA button fill, active nav item indicator, lead state "in_sequence" badge, keyboard shortcut key highlight, approve button background |
| Destructive | `oklch(55% 0.18 25)` — deep rose-red | Delete lead confirmation button, "mark do-not-contact" destructive action only |
| Text primary | `oklch(20% 0.008 60)` — near-black with warm tint | Body text, headings |
| Text secondary | `oklch(48% 0.008 60)` — warm mid-gray | Metadata, timestamps, labels |
| Border | `oklch(88% 0.010 55)` — warm light border | Card borders, dividers |
| Health red | `oklch(55% 0.20 25)` — vivid red | Integration health card failure state ONLY |
| Health green | `oklch(60% 0.14 145)` — warm green | Integration health card healthy state |

### Glass Card Treatment

All cards use the glass/frosted pattern declared in CLAUDE.md:

```
backdrop-blur-md bg-white/10 (light mode)
backdrop-blur-md bg-white/5 (dark mode)
border border-white/10
shadow-[inset_0_1px_0_rgba(255,255,255,0.08)]
```

Custom swappable backgrounds are set via `--bg-image` on `:root`. Cards must remain legible against any background image (the `backdrop-blur-md` ensures this).

**Source:** CLAUDE.md (glass cards, warm colors, dark/light toggle, --bg-image); impeccable skill (OKLCH, no #000/#fff, tinted neutrals); design-taste-frontend skill (no neon, no purple AI aesthetic).

Accent reserved for: CTA button fills, active nav state, "in_sequence" lead state badge, keyboard shortcut highlights, approve button. Never used as a general hover color or decorative accent.

## Glass Card Anatomy (Locked Pattern)

Every card surface in Phase 1 follows this structure:

```
rounded-2xl
backdrop-blur-md
bg-white/10 (light) / bg-white/5 (dark)
border border-white/10
shadow-[inset_0_1px_0_rgba(255,255,255,0.08)]
p-6 (lg spacing)
```

No nested cards. No side-stripe borders. No gradient text. No identical card grids.

## Interaction Contracts

### Keyboard Navigation (Draft Queue)

| Key | Action |
|-----|--------|
| A | Approve current draft, advance to next |
| S | Skip (defer) current draft, advance to next |
| H | Hold current draft, advance to next |
| Esc | Exit keyboard mode, return to list |

Keyboard mode activates only when a DraftCard is focused (click or tab into queue). Visual indicator: ring on focused card, keyboard shortcut badges pulse once on activation.

### Motion Contracts

All Framer Motion usage follows the spring standard: `type: "spring", stiffness: 120, damping: 18`.

| Interaction | Animation |
|-------------|-----------|
| Page load — lead list rows | Staggered fade-up, `staggerChildren: 0.04s` |
| Draft card approve/skip/hold | Slide-left exit, next card slide-right enter, `layoutId` for smooth transition |
| Integration health card expansion | Height animate, `layout` prop |
| Lead state badge change | Color cross-fade via Framer Motion `animate` |
| Sheet open (AddLead, CreateCoach) | Radix Sheet built-in animation — do not override |
| Toast notifications | Slide up from bottom, auto-dismiss 4s |
| Skeleton loaders | Shimmer: `animate={{ backgroundPosition: ["200% 0", "-200% 0"] }}` on gradient, 1.5s loop |

Never animate CSS layout properties (`top`, `left`, `width`, `height`). Animate `transform` and `opacity` only.

### Loading States

Every async operation has a skeleton loader that matches the target layout geometry. No generic spinners. Skeletons use the warm border color as base, shimmer animation.

### Error Boundaries

Every major section (LeadListPage, LeadProfilePage, AdminDashboard, DraftQueueScaffold) is wrapped in a React error boundary. Error boundary UI: card with Phosphor `WarningOctagon` icon, error summary, "Try again" button. No raw stack traces visible to coaches.

### Empty States

Every list and table has a designed empty state (not a blank area). See Copywriting Contract below.

### Error States

| Trigger | Message |
|---------|---------|
| Gmail OAuth failure (health card) | "Gmail disconnected. Your sequences are paused. Reconnect to resume." |
| Lead save failed | "Couldn't save this lead. Check your connection and try again." |
| Coach notes save failed | "Notes couldn't be saved. Your changes are still here — try again." |
| Draft action failed | "This action didn't go through. Refresh and try again." |
| Admin invite failed | "Invite couldn't be sent. Check the email address and try again." |
| Auth: invalid invite token | "This invite link has expired or already been used. Ask Daniel for a new one." |
| Auth: wrong credentials | "Incorrect email or password. Try again." |

### Destructive Confirmations

| Action | Trigger | Confirmation copy |
|--------|---------|------------------|
| Remove lead | "Remove lead" button on profile | Dialog: "Remove [Lead Name]? This will delete their profile, timeline, and any pending drafts. This cannot be undone." / Confirm: "Yes, remove" / Cancel: "Keep lead" |
| Mark do-not-contact | State override to `do_not_contact` | Inline confirmation: "Mark [Lead Name] as do-not-contact? No further emails will ever be sent to this address." / Confirm: "Mark do-not-contact" |

No generic "Are you sure?" copy. Every confirmation names the specific lead and states the consequence.

### Tone Notes

Premium throughout. No placeholder text visible to coaches. Voice: direct, warm, helpful. No startup jargon ("streamline", "elevate", "seamless", "next-gen", "unleash"). No em dashes in any copy. Use colons, commas, and periods instead.

The product is a managed service run by Daniel — coaches should feel they're using a considered tool built specifically for them, not a generic SaaS.

## Responsive Behavior

| Breakpoint | Behavior |
|-----------|---------|
| < 768px (mobile) | Single-column layout. Sidebar collapses to bottom nav or hamburger. LeadProfile stacks vertically. Draft editor full-width. |
| 768px–1024px (tablet) | Two-column with narrower sidebar (60px icon-only). Lead profile: stacked. |
| > 1024px (desktop) | Full two-column layout. 240px sidebar. LeadProfile: 60/40 split. |

Use `min-h-[100dvh]` — never `h-screen`. Grid over flexbox percentage math. `max-w-[1400px] mx-auto` on page container.

## Accessibility

- All interactive elements reachable by keyboard.
- Focus rings use the accent color: `ring-2 ring-accent ring-offset-2`.
- Color is never the sole differentiator (badges include text labels, health card includes icon + text).
- Touch targets: minimum 44px height on all rows, buttons, and nav items.
- ARIA labels on icon-only buttons (ThemeToggle, close buttons, keyboard shortcut pills).
- Screen reader announcements for draft approval transitions (live region).
