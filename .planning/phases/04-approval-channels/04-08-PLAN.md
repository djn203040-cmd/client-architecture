**Locate the lifecycle transition.** Read `apps/web/app/api/drafts/generate/route.ts` end-to-end. The current draft-creation flow is:

1. Insert a draft row with `status='generating'` (~line 87).
2. Fire-and-forget block calls `generateDraft(...)` (~line 109).
3. On success, UPDATE the draft row with `body`, `status='pending'`, `confidence_level`, `generation_context` (~line 162-175).

The branching lands at step 3. After the existing UPDATE — and inside the same fire-and-forget block so we don't slow the HTTP response — read `coaches.autonomous_mode` and `coaches.scheduled_send_at` defaults, then fire the appropriate Inngest event(s).

**Implementation pattern:**

```typescript
// Inside the existing fire-and-forget block (~line 107), right after the existing
// `await supabase.from('drafts').update({ ..., status: 'pending', ... })` call (~line 162-175).
//
// IMPORTANT: do NOT touch the existing UPDATE call directly when autonomous_mode is 'manual'/null —
// the existing UPDATE writes status='pending' which is the correct terminal status for both
// 'manual' and 'mode_b' modes. ONLY Mode A needs to overwrite that status to 'approved'.

import { inngest } from "@/inngest/client";  // add near other imports if not present

// ... existing UPDATE that sets status='pending' (DO NOT REMOVE) ...

// Read autonomous_mode and the scheduled send time
const { data: coachWithMode } = await supabase
  .from('coaches')
  .select('autonomous_mode')
  .eq('id', coachId)
  .single();

const { data: draftRow } = await supabase
  .from('drafts')
  .select('id, scheduled_send_at, coach_id, lead_id, body, subject, confidence_level')
  .eq('id', draftId)
  .single();

// Resolve lead name for notification payload
const { data: leadRow } = await supabase
  .from('leads')
  .select('name, email')
  .eq('id', leadId)
  .single();

const mode = (coachWithMode?.autonomous_mode ?? 'manual') as 'manual' | 'mode_a' | 'mode_b';

if (mode === 'mode_a') {
  // Mode A: skip the queue. Overwrite status to 'approved' and fire Gmail send.
  // The 'pending' status set by the line above is intentionally short-lived for Mode A.
  await supabase
    .from('drafts')
    .update({ status: 'approved', approved_at: new Date().toISOString() })
    .eq('id', draftId)
    .eq('status', 'pending');  // CAS guard — only flip if still pending (defense vs. coach manual action mid-window)

  await inngest.send({
    name: 'draft/send_via_gmail',
    data: { draftId, coachId, source: 'mode_a' },
  });
  // NOTE: intentionally do NOT fire notification/draft_ready — Mode A drafts skip the queue
} else if (mode === 'mode_b') {
  // Mode B: leave status='pending' and schedule the auto-send timer.
  // The dashboard still surfaces the draft; the coach can approve/hold manually before the timer wakes.
  // autonomousModeBTimer cancels on draft/approved_manually / draft/held_manually / draft/cancelled.
  await inngest.send({
    name: 'draft/created_mode_b',
    data: {
      draftId,
      coachId,
      scheduledSendAt: draftRow?.scheduled_send_at ?? new Date(Date.now() + 24 * 60 * 60 * 1000).toISOString(),
    },
  });
} else {
  // Manual (default): pending status remains; fire follow-up timer + notification fan-out
  await inngest.send({
    name: 'draft/created_pending',
    data: {
      draftId,
      coachId,
      createdAt: new Date().toISOString(),
    },
  });
  await inngest.send({
    name: 'notification/draft_ready',
    data: {
      coachId,
      eventType: 'draft_ready',
      payload: {
        draftId,
        leadName: leadRow?.name ?? 'your lead',
        leadEmail: leadRow?.email ?? undefined,
        subject: draftRow?.subject ?? undefined,
        body: draftRow?.body ?? undefined,
        sendTime: draftRow?.scheduled_send_at ?? undefined,
        confidenceLevel: (draftRow?.confidence_level === 'low' ? 'low' : 'high'),
      },
    },
  });
}
```

**Edge cases:**

- Draft generation failed (status='error' set elsewhere ~line 158): do NOT fire any event. Skip the entire branching block when status is not 'pending' at read time.
- `coaches.autonomous_mode` column is NULL: treat as 'manual' (default).
- `draftRow.scheduled_send_at` is NULL: fall back to `now() + 24h` (so the Mode B timer at least sleeps a reasonable amount).
- The fire-and-forget block already has a try/catch ~line 198. Wrap the Inngest calls inside the same try so a transient Inngest error does not crash the draft creation.
