| coach browser -> /api/leads/[id]/thread | Coach requests a lead's email thread |
| thread route -> Gmail API | Coach OAuth token used to read Gmail data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-24 | Spoofing | Unauthenticated thread fetch | mitigate | `supabase.auth.getUser()` 401 gate |
| T-02-25 | Information Disclosure | Coach reads another coach's lead thread | mitigate | Ownership check: `lead.coach_id === user.id` else 403; thread fetched only with the requesting coach's own Gmail client |
| T-02-26 | Information Disclosure | Gmail thread of a lead not belonging to the coach | mitigate | thread route resolves threadId from `email_events` scoped to the verified lead_id; `getGmailClientForCoach(user.id)` only reads the coach's own mailbox |
| T-02-27 | Tampering | Malicious HTML in an email body rendered as markup | mitigate | Body rendered as plain text via `whitespace-pre-wrap` (text node), not `dangerouslySetInnerHTML` — no script execution |
| T-02-28 | Information Disclosure | Email content in server logs | accept | No email body logged; low volume at launch |
</threat_model>
