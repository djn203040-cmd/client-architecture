## Spacing Scale

Declared values — multiples of 4 only. All values from established DESIGN.md rhythm.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline chip gap |
| sm | 8px | Compact element spacing, chip internal padding |
| md | 16px | Default element spacing, card section gaps |
| lg | 24px | Card internal padding (standard), section separation |
| xl | 32px | Between major card stacks |
| 2xl | 48px | Empty state vertical centering padding |
| 3xl | 64px | Page-level breathing room |

**Exceptions:**
- Touch targets: all interactive elements minimum 44px height/width (`min-h-[44px] min-w-[44px]`). Applies to chip remove buttons, regen button, assign button, expand email row.
- Voice corpus textarea: minimum height 128px (8 lines) to encourage real paste behavior, not token entry.
- AI lead description card: pinned above tabs — internal padding `p-4` (not `p-6`) to stay compact and always visible without consuming excessive vertical space.

## Color

Full palette carried from DESIGN.md (OKLCH). No new colors introduced for Phase 2.

| Role | Light | Dark | Usage |
|------|-------|------|-------|
| Dominant (60%) — `--background` | `oklch(97% 0.008 60)` | `oklch(14% 0.010 50)` | Page surface, settings page background |
| Secondary (30%) — `--card` | `oklch(94% 0.010 55)` | `oklch(19% 0.012 50)` | Standard card fill, settings section containers |
| Accent (10%) — `--accent` / `--primary` | `oklch(62% 0.14 50)` | `oklch(70% 0.14 50)` | Amber-gold: reserved for specific elements listed below |
| Destructive — `--destructive` | `oklch(55% 0.18 25)` | `oklch(65% 0.18 25)` | Remove chip (never-say list), remove example row |
| Confidence warning — amber | `oklch(72% 0.12 70)` bg / `oklch(40% 0.10 65)` text | `oklch(68% 0.12 68)` bg / `oklch(85% 0.08 65)` text | Voice confidence warning badge only. Warm amber, NOT --destructive red. |
| Glass surface | `bg-white/10 dark:bg-white/5 border border-white/10 shadow-[inset_0_1px_0_rgba(255,255,255,0.08)]` + `backdrop-blur-md` | same | Priority elevated surfaces (see Elevation section) |

**Accent (`--accent`) is reserved for exactly these elements — nothing else:**
1. Primary CTA buttons (Generate Draft, Analyze Corpus, Assign Transcript)
2. Active tab indicator on lead profile tab bar (Thread | Timeline | Notes)
3. Active chip "confirmed" state in voice profile card (ring, not fill)
4. Focus rings on all interactive elements (`focus:ring-accent`)
5. Re-analyze button (primary, only one per voice builder section)

**Accent is NOT used for:** never-say chip borders, example row hover states, inline edit save, confidence warning badges, "AI-generated" label, email expand/collapse chevrons.

**Confidence warning badge color contract:**
- Background: warm amber `oklch(72% 0.12 70)` in light / `oklch(25% 0.08 65)` in dark
- Text: `oklch(40% 0.10 65)` in light / `oklch(85% 0.08 65)` in dark
- Icon: `WarningCircle` from Phosphor at `size-3.5`
- No border. Subtle fill only.
- This is the same amber treatment used on both the DraftCard confidence badge (existing) and the new voice builder confidence warning.

**AI-generated indicator:** Small inline label using `--muted-foreground` text, `text-[10px]`, no background. Text: "AI-written". Sits below the AI lead description paragraph, right-aligned.

## Interaction Contracts

### Voice Builder Flow

1. Coach lands on `Settings → My Voice` — sees VoiceCorpusImporter in idle state with empty sections
2. Coach pastes content into one or more channels — "Analyze my writing" button appears
3. Coach clicks "Analyze" — button shows spinner; sections remain visible and frozen (no loading overlay)
4. On complete — VoiceCorpusImporter scrolls down / transitions to show VoiceProfileCard + ExamplesList below it (no page navigation, same settings page scroll)
5. Coach interacts with chips — remove, add, confirm — changes are local state until explicit "Save profile" button (secondary, below cards)
6. "Save profile" commits to `coaches.voice_model` via `POST /api/voice/save`
7. If fewer than 8 examples: confidence warning visible below ExamplesList at all times until resolved

### Transcript Unmatched Queue

1. Coach opens Drafts view — sees tab bar with "Drafts | Unmatched" (count badge on Unmatched when > 0)
2. Unmatched tab shows transcript rows with fuzzy suggestions when available
3. One-click "Yes, assign" on fuzzy suggestion: row disappears, toast "Assigned to [Lead Name]. Draft generating..."
4. Manual assign: coach types in Command combobox, selects lead, clicks "Assign" — same toast
5. After assign: row removed from unmatched queue via optimistic removal + server confirmation

### Lead Profile Tab Navigation

1. Lead profile page loads with Thread tab active (leftmost)
2. Most recent email auto-expanded, older emails collapsed
3. Click collapsed email row — expands inline with `height: auto` transition (`transition-all duration-200`)
4. Timeline tab: existing ActivityTimeline component (no changes)
5. Notes tab: existing CoachNotesField component (no changes)
6. AI Lead Description card is ABOVE the tab bar — always visible regardless of active tab. Not in a tab.

### Generate Draft Interaction

1. Coach clicks "Generate draft" on lead profile — button enters generating state immediately
2. `POST /api/drafts/generate` returns `{ draftId, status: 'generating' }` within ~200ms
3. Button stays in generating state until Supabase Realtime notifies of draft status change to `pending`
4. On `pending`: button disappears, toast "Draft ready — check your queue"
5. Hard-blocked leads: button not rendered at all. No disabled state, no tooltip. Just absent.

## Accessibility

All elements must meet WCAG 2.1 AA:

| Requirement | Implementation |
|-------------|---------------|
| Focus management | All interactive elements reachable via Tab. DraftCard auto-focuses on mount (existing pattern). Inline edit textarea auto-focuses when activated. |
| Touch targets | 44px minimum on all buttons (enforced via `min-h-[44px] min-w-[44px]`) |
| Screen reader labels | `aria-label` on icon-only buttons (regen, chip remove, email expand). `aria-live="polite"` on draft count, unmatched count. |
| Keyboard navigation | Command combobox (lead picker) fully keyboard-navigable via shadcn Command. Email rows keyboard-expandable via Enter/Space. |
| Color independence | Confidence warning uses icon + text + color (not color alone). AI-written indicator uses text label (not icon alone). |
| Reduced motion | `prefers-reduced-motion: reduce` — disable chip reflow animation and VoiceProfileCard entry animation. Spring animations on DraftCard already respect this via Framer Motion defaults. |
