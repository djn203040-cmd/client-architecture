<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fireflies/Zoom -> /api/webhooks/transcripts/* | Untrusted external POST; only HMAC proves authenticity |
| webhook handler -> transcripts table | adminClient bypasses RLS — coach scoping must be enforced in code |
| coach browser -> /api/transcripts/* | Coach-supplied transcript text crosses into server |
| transcript content -> AI engine context | Lead/coach-supplied text becomes LLM input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Spoofing | Forged Fireflies/Zoom webhook POST | mitigate | HMAC-SHA256 verification with `timingSafeEqual`; 401 before any DB write (TRANS-007) |
| T-02-08 | Elevation of Privilege | adminClient writes transcript to wrong coach | mitigate | Resolve coachId from URL (Fireflies) / integrations table (Zoom); verify coach exists before insert; matchTranscriptToLead scopes leads query to coach_id |
| T-02-09 | Tampering | Replay of a captured valid webhook | mitigate | Zoom timestamp folded into signed message; Fireflies meetingId dedupe — skip insert if a transcript with the same provider+meetingId exists |
| T-02-10 | Tampering | Prompt injection embedded in transcript text | mitigate | Downstream AI engine (02-03) wraps transcript in XML delimiters; system prompt instructs facts-only |
| T-02-11 | Spoofing | Unauthenticated /api/transcripts or /assign call | mitigate | `supabase.auth.getUser()` 401 gate; ownership `.eq("coach_id", user.id)` check on assign |
| T-02-12 | Information Disclosure | Transcript PII in logs | accept | No transcript content logged; low volume at launch |
</threat_model>
